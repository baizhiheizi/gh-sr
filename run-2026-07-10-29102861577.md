## Discovered contracts (for future reference)
- **`extractTrailingPercent` implicitly tolerates leading whitespace** (" 3.2%" → 3.2, not 0). The walk-back loop stops at any non-digit/non-dot character — including the leading space — so `strconv.ParseFloat` gets a clean "3.2" slice. This is implicit; a future port to `strings.Index` semantics that returns 0 here would be a behaviour change flagged by `TestExtractTrailingPercent_edgeCases/leading_space_before_digit`.
- **`tui.updateFilterList` extraction** preserves "choices read at call time" semantics by NOT capturing the slice in the closure argument — important: if the dashboard re-enters the filter panel between key presses, `filterHostChoices`/`filterRepoChoices` are refreshed by the upstream menu handler, and the deduped helper sees the latest values.
- **Test sub-cases with `/` in the name get auto-escaped** to `_` by Go's `t.Run(tc.name, ...)` (e.g. `"full used/total cell"` → `full_used/total_cell` in the PASS line).
