## Discovered contracts (for future reference)
- **`ensureDoctorHostOS` pre-set short-circuit is critical** — without it, a remote host with a config-file OS override would silently re-detect via SSH, adding one round-trip to every doctor run and risking drift if the config disagrees with the live host.
- **`checkLinuxSudo` id-u-error must NOT proceed to sudo probe** — otherwise operators see misleading "passwordless sudo not available" when the real problem is SSH connectivity.
- **`checkNativeRunnerInstall` skips non-matching hosts by host name** — the predicate lives in `installTargetsForHost`. Don't try to short-circuit inside the loop; the existing `installTargetsForHost` filter is the right level.
- **Test pinning the SSH-call count** in `checkLinuxSudo` (`calls != 2`) is a useful invariant — a future refactor that splits the probe into two or folds them into one SSH round-trip will need to update this assertion, which is exactly when the maintainer should review the optimization.
